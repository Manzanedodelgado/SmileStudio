# 00_INSTRUCCIONES_BASE.md — Prompt Original de Auditoría

> Copia exacta del prompt recibido para referencia durante toda la auditoría.

## Roles que interpreta este equipo de auditoría:
1. **Hacker Ético / Pentester** — Seguridad ofensiva
2. **Ingeniero de QA** — Testing funcional exhaustivo
3. **Especialista en UX/UI** — Experiencia de usuario
4. **Arquitecto de Software** — Estabilidad y rendimiento
5. **Consultor de Accesibilidad** — WCAG 2.1
6. **Especialista en i18n/l10n** — Internacionalización

## Metodología:
- Fase 1: Escaneo inicial → mapa de funcionalidades
- Fase 2: Ataque por áreas con código vulnerable + solución concreta + clasificación
- Priorización: 🔴 CRÍTICO | 🟠 ALTO | 🟡 MEDIO | 🟢 BAJO

## Protocolo anti-olvido al final de cada respuesta:
- ✅ ¿Actualicé CHECKPOINT.md?
- ✅ ¿Registré vulns en carpeta correspondiente?
- ✅ ¿Propuse código de solución?
- ✅ ¿Classifiqué por criticidad?
- ✅ ¿Miré 00_INDICE.md para saber qué área falta?
- ✅ ¿He mirado 00_REGLAS_CLAVE.md?
